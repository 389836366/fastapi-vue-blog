# 7月17日理论作业

### 1

DOM大概是什么？它解决了什么问题？

回答：DOM 是浏览器把 HTML 页面转换成的一组 JavaScript 可以操作的对象。
 它让 JavaScript 能够查找、读取和修改网页中的元素与内容。

### 2

下面代码中的 `document` 表示什么？

```javascript
document.querySelector("#blog-title");
```

回答：document表示当前网页的意思。

### 3

`querySelector()`有什么作用？

回答：表示按照选择器去寻找元素。

### 4

为什么这里需要写 `#`？

```javascript
document.querySelector("#blog-title");
```

下面两个选择器分别寻找什么？

```javascript
"#blog-title"
".post-title"
```

回答：因为要选中id。寻找id名为blog-title的元素。寻找类选择器post-title的元素。

### 5

下面两段代码输出的内容有什么区别？

```javascript
console.log(blogTitleElement);
```

```javascript
console.log(blogTitleElement.textContent);
```

回答：第一段代码的意思是输出的是一个完整的 DOM 元素对象，可以在控制台中看到它对应的 HTML 标签、属性和内容。第二段代码的意思是只输出blogTitleElement里面的文本内容。

### 6

下面代码有什么作用？

```javascript
blogDescriptionElement.textContent = "新的介绍";
```

回答：修改blogDescriptionElement里面的文本内容，改为“新的介绍”这四个字。

### 7

如果 `querySelector()`没有找到元素，通常会得到什么？

回答：null

### 8

为什么你的 `<script src="main.js"></script>` 放在 `</body>` 前面，有利于JavaScript找到前面的HTML元素？

回答：浏览器通常按照从上到下的顺序解析 HTML。脚本放在 `</body>` 前时，前面的 HTML 元素一般已经创建完成，所以 JavaScript 能够找到并操作它们。如果脚本执行得太早，元素还没有创建，`querySelector()` 就可能得到 `null`。

### 9

页面中有多个 `.post-title a` 时，下面代码会返回几个元素？

```javascript
document.querySelector(".post-title a");
```

回答：第一个
