# 第一关：基础概念

## 1

下面的代码有什么作用？

```css
.header-container {
  display: flex;
}
```

它会直接控制 HTML 中的哪些元素？

回答：它的作用是让header-container这里面的元素按照flex的方式呈现。主要控制HTML中的site-brand元素和navbar元素。

## 2

```css
justify-content: space-between;
```

在你的页眉中产生了什么效果？

如果把它删除，页面可能发生什么变化？

1.回答：让我页眉里面的内容两端对齐。

2.回答：如果删除，两部分会向左排列。

## 3

下面两个属性分别控制什么？

```css
gap: 30px;
align-items: center;
```

回答：第一行的gap是控制项目间距，第二行的意思控制交叉轴对齐，这里是居中。

## 4

```css
flex-wrap: wrap;
```

有什么作用？在哪种情况下比较容易观察到它的效果？

回答：他的作用是控制是否换行。在改变浏览器界面大小时就可以看到它的效果，当父容器的可用宽度不足，无法在一行容纳所有子元素时，后面的子元素会换到下一行。



# 第二关：HTML与CSS关系

## 5

解释下面两段代码为什么能够配合工作：

```html
<a href="#about">关于我</a>
```

```html
<section id="about" class="about">
```



其中：

- `href="#about"` 的作用是什么？
- `id="about"` 的作用是什么？
- `class="about"` 的作用是什么

回答：1.href="#about"的作用是点击可以跳转到id为about的位置上。2. id="about"的作用是给元素设立一个唯一的标志名叫about。3. class="about"的作用是给这个元素取一个类名叫about。

## 6

下面这个选择器会选中哪些元素？

```css
.nav-list a {
  color: white;
}
```

它会不会影响页面里的所有链接？为什么？

回答：不会，因为这个后代选择器的意思是选中nav-list里面的a，而不是整个页面的a。

## 7

下面的代码什么时候生效？

```css
.nav-list a:hover {
  color: #ffd166;
}
```

`:hover` 是什么意思？

回答：当鼠标放到链接上面就会生效。:hover的意思是让鼠标悬停到指定元素上面可以实现指定的效果。

# 第三关：Flex父子关系

观察你自己的 HTML：

```html
<div class="post-list">
  <article class="post-card">...</article>
  <article class="post-card">...</article>
  <article class="post-card">...</article>
</div>
```

## 8

为什么 `display: flex` 应该写在 `.post-list` 上，而不是分别写在三个 `.post-card` 上？

回答：`display: flex` 会让一个元素成为 Flex 容器，并排列它的直接子元素。

## 9

如果删除：

```css
.post-list {
  display: flex;
}
```

三张文章卡片大概会变成什么排列方式？

回答：上下排列而不是横向排列。

## 10

解释下面这行代码的大概作用，不要求解释得特别专业：

```css
flex: 1 1 200px;
```

它为什么能让三张文章卡片根据页面空间调整宽度？

回答：这实际上是三个属性的简写：等价于：

```css
flex-grow: 1;
flex-shrink: 1;
flex-basis: 200px;
```

分别表示：

### `flex-grow: 1`

空间有剩余时，卡片允许变宽。

三张卡片都是 `1`，因此会较平均地分配剩余空间。

### `flex-shrink: 1`

空间不足时，卡片允许缩小。

### `flex-basis: 200px`

在分配空间之前，卡片的基础宽度大约按照 `200px` 计算。



连起来理解就是：

每张卡片先以 `200px` 作为基础宽度；空间充足时一起变宽，空间不足时可以缩小；实在放不下时，因为父元素设置了 `flex-wrap: wrap`，卡片就会换行。
