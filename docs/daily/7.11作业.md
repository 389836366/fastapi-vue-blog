# 7.11理论题

## 1


为什么访问：http://localhost:5500/frontend/ 时，浏览器会自动显示`index.html`？

回答：当浏览器访问一个目录而没有指定文件时，Live Server会按照默认首页规则寻找 `index.html`。如果目录中存在这个文件，就自动返回它。

## 2

解释：

```html
<a href="article.html">我的第一篇博客</a>
```

其中 `<a>` 和 `href` 分别负责什么？

回答：<a>是创造一个链接，href是指定这个链接去哪。

## 3

`index.html`、`article.html` 和 `style.css` 位于同一个文件夹，为什么可以写：

```html
<link rel="stylesheet" href="style.css">
```

回答：`article.html` 和 `style.css` 位于同一目录，所以可以通过相对路径 `style.css` 找到样式文件。

## 4

下面两个选择器有什么区别？

```css
h1
```

```css
.article-detail h1
```

它们各自会影响哪些标题？

回答：第一个代码会影响全部的h1标签。第二个代码只影响article-detail里面的h1.

## 5

你的发布时间是：

```html
<p class="article-meta">发布时间：2026-07-08</p>
```



它会不会同时受到下面两段CSS影响？为什么？

```css
.article-detail p {
  line-height: 1.8;
}
```

```css
.article-meta {
  color: #888;
}
```

回答：会。因为CSS不是“一个元素只能匹配一个选择器”，一个元素可以同时匹配很多条规则。

## 6

为什么给 `.back-link` 添加：

```css
display: inline-block;
```

以后，更方便设置上下间距？

回答：因为<a> 默认属于行内元素。行内元素主要用于文字内部，上下外边距通常不能像普通盒子一样稳定地撑开布局。设置后它仍然可以和文字一样排列，同时又能更完整地使用：margin，padding，width，height。

## 7

解释下面三段代码如何配合把页脚推到底部：

```css
body {
  min-height: 100vh;
  display: flex;
  flex-direction: column;
}
```

```css
main {
  flex: 1;
}
```

回答：min-height: 100vh是让 `body` 最小高度等于整个浏览器窗口。display: flex;和flex-direction: column是让`header`、`main`、`footer` 从上到下排列。flex: 1是让`main` 占据页眉和页脚之间的剩余空间，从而把 `footer` 推到窗口底部。

## 8

为什么把 `transition` 写在：

```css
.post-title a
```

里面，比只写在：

```css
.post-title a:hover
```

里面更合适？

回答：因为写在:hover通常只能保证进入悬停状态时使用过渡，鼠标移出时可能立即恢复。
