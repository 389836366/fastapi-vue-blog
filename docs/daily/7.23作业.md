# 7.23作业

1. `document.createElement()`有什么作用？

   回答：`document.createElement()`用于创建一个新的HTML元素对象。这个元素刚创建时只存在于JavaScript内存中，还没有加入DOM页面。

2. 创建出来的元素会立即显示在页面上吗？为什么？

   回答：不会。刚创建的元素只存在于JavaScript内存中，还没有被添加到页面的DOM结构里。需要使用`appendChild()`等方法将它加入页面后才会显示。

3. `appendChild()`有什么作用？

   回答：`父元素.appendChild(子元素)`会把括号中的子元素添加到父元素内部，并放在已有子元素的最后面。

4. `createArticleCard(article)`中的`article`代表什么？

   回答：`article`是`createArticleCard()`的形参，代表当前传入的一篇文章对象。在循环中，它会依次代表数组中的第一篇、第二篇和第三篇文章。

5. `createArticleCard()`为什么需要`return articleElement`？

   回答：`return articleElement`把创建并组装完成的文章卡片返回给调用函数的位置。这样`renderArticleList()`才能接收到这张卡片，并使用`appendChild()`把它添加到页面中。如果不写`return`，函数调用得到的是`undefined`。

6. `createArticleCard()`和`renderArticleList()`分别负责什么？

   回答：`createArticleCard(article)`接收一篇文章对象，创建并组装一张文章卡片，最后返回卡片。
    `renderArticleList(articleList)`接收整个文章数组，遍历数组，为每篇文章调用一次`createArticleCard()`，再把得到的卡片添加到页面中。

7. `postListElement.textContent = ""`有什么作用？

   回答；这行代码把`postListElement`内部的所有旧内容清空。这样重新渲染文章列表时，不会在原有卡片后面重复添加卡片。

8. `for...of`在渲染文章列表时做了什么？

   回答：`for...of`依次取出`articleList`数组中的每一篇文章，并将当前文章保存到`article`中。循环体再把`article`传给`createArticleCard()`创建卡片，最后把卡片添加到`postListElement`中。

9. 数组有3个元素时，`createArticleCard()`为什么执行3次？

   回答：因为articleList里面的元素数量实际上是articles的数量，articleList里面的元素一个个放进article里面，因为数组有3个，所以执行三次。

10. 为什么`document.querySelector(".post-title")`必须在渲染列表后执行？

    回答；动态文章卡片一开始不存在于DOM页面中，只有执行`renderArticleList(articles)`后，JavaScript才会创建并添加`.post-title`元素。如果提前查询，页面中还没有这个元素，`querySelector()`会得到`null`。

11. 静态手写卡片和根据数组动态创建卡片有什么区别？

    回答：静态手写是把内容**写死**在 HTML 里，适合内容不会变的页面。动态创建是把数据放在数组里，用 JS **循环自动生成**，加内容只改数据、改结构只改模板，适合内容会变化的页面。

12. 增加第四篇文章后，为什么不需要再手写一张HTML卡片？

    回答：因为我们的两个函数本质就是手写HTML卡片的功能，因此我们增加第四篇文章后把它加入到数组就行了。

13. `isPublished`如何决定状态文字和CSS类？

    回答：程序使用`if...else`判断`article.isPublished`。如果它是`true`，状态文字设置为“状态：已发布”，并添加`status-published`类；如果是`false`，状态文字设置为“状态：草稿”，并添加`status-draft`类。

14. 完整解释从`articles`数组到页面显示3张卡片的执行过程。

    回答：首先调用`renderArticleList(articles)`，把`articles`数组传给形参`articleList`。函数先使用`postListElement.textContent = ""`清空旧卡片，然后通过`for...of`依次取出数组中的每一篇文章。
     每次循环都会调用`createArticleCard(article)`。这个函数创建`article`、标题、简介、日期、分类和状态元素，从当前文章对象中读取数据，并根据`isPublished`设置状态文字和CSS类。然后使用`appendChild()`把这些元素组装进`articleElement`，最后通过`return articleElement`返回完整卡片。
     `renderArticleList()`接收到返回的卡片后，再用`postListElement.appendChild(articleCardElement)`把卡片添加到页面。数组中有3篇文章，因此循环执行3次，最终页面显示3张卡片。
